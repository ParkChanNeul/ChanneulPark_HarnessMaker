# Vertical Slice Preview: Change or Cancel an Appointment

## Teacher Review Status

- Scope: `partial`
- Vocabulary Set: `proposed`
- Default duration: 90 minutes
- Supported duration: 60–120 minutes
- Student Deck / HTML: 생성하지 않음

## Lesson Promise

몸 상태를 짧게 말하고 예약이나 약속을 변경 또는 취소한 뒤 새로운 시간을 확인할 수 있다.

## Core Student Chain

이 Chain은 Preview와 이후 Practice에서 실제 산출 대상으로 사용한다.

1. `안녕하세요.`
2. `몸이 안 좋아요.`
3. `예약을 바꾸고 싶어요.`
4. `내일 오후는 가능하세요?`
5. `네, 세 시로 바꿔 주세요.`
6. `갑자기 연락해서 죄송합니다.`
7. `감사합니다.`

기본 수업에서는 `몸이 안 좋아요.`와 `예약을 바꾸고 싶어요.`를 두 문장으로 유지한다. `몸이 안 좋아서 예약을 바꾸고 싶어요.`로 합치는 활동은 `grammar_reason_eoseo`가 승인된 120분 확장에서만 사용한다.

## Proposed Language Target Use

| Target | Treatment | Preview에서 보이는 기능 |
|---|---|---|
| `grammar_want_go_sipeoyo` | new | `바꾸고 싶어요`, `취소하고 싶어요` |
| `grammar_request_verb_eo_juseyo` | review | `세 시로 바꿔 주세요` |
| `register_haeyo_polite` | retrieval | 짧은 해요체 시작·응답·마무리 |
| `interaction_adjust_appointment` | practice | reschedule, cancel, confirm_time |
| `interaction_apologize` | practice | `갑자기 연락해서 죄송합니다` |
| `interaction_confirm_and_answer` | practice | `내일 오후는 가능하세요?` / `네, 세 시는 가능해요` |
| `grammar_reason_eoseo` | defer | 60/90분에서는 분석·생산하지 않음 |

## 60분 구성

| 시간 | 단계 | 학생 행동 |
|---|---|---|
| 0–5 | Previous lesson retrieval | `아파요`, `오늘 아침`, 짧은 해요체를 모델 없이 한 번 회수 |
| 5–12 | Mission and chain | Core Student Chain을 듣고 `change / cancel / confirm` 기능 구분 |
| 12–22 | Vocabulary core | `예약, 바꾸다, 취소하다, 가능하다`를 결합 표현으로 연습 |
| 22–32 | New grammar | `예약을 바꾸고 싶어요`와 `예약을 취소하고 싶어요` 대조 |
| 32–42 | Guided Practice | 제시된 날짜·시간 카드로 질문하고 응답 |
| 42–54 | Independent Roleplay | reschedule 또는 cancel 중 하나를 선택해 짧은 통화 완성 |
| 54–60 | Exit Check | 이유 한 문장, 변경/취소 한 문장, 새 시간 확인 한 문장 산출 |

### 60분 압축 규칙

- Productive Core 6개 중 `예약, 바꾸다, 취소하다, 가능하다`를 우선한다.
- `약속, 연락하다`는 Chain 안에서 모델로 제공하되 확장 Drill은 생략한다.
- Receptive Support는 직원 발화에서만 제공한다.
- `grammar_reason_eoseo`는 계속 defer한다.

## 90분 구성

| 시간 | 단계 | 학생 행동 |
|---|---|---|
| 0–10 | Evidence-aware warm-up | 이전 노출 표현을 회수하고 실제 숙련 여부를 짧게 확인 |
| 10–20 | Mission map | 몸 상태 → 변경/취소 → 새 시간 확인 → 사과/마무리 순서 이해 |
| 20–35 | Vocabulary in combinations | 6 Productive Core와 4 Receptive Support를 문장·Q/A로 연결 |
| 35–50 | Controlled to Guided | `V-고 싶어요`, `V-아/어 주세요`, 가능 시간 질문을 단계적으로 생산 |
| 50–65 | Guided appointment practice | reschedule, cancel, confirm_time Prompt에 맞춰 다음 문장 선택 |
| 65–80 | Independent Roleplay | 예약 변경 1회와 취소 1회 수행; 모델은 핵심 명사만 제공 |
| 80–87 | Messaging/call transfer | open → request → close 순서로 같은 Chain을 문자/전화에 적용 |
| 87–90 | Exit Check | Core Chain 중 교사 Prompt 없이 나온 문장과 지원이 필요했던 문장 기록 |

## 120분 확장 구성

90분 Core를 유지하고 다음 30분만 추가한다.

| 시간 | 확장 | 학생 행동 |
|---|---|---|
| 90–100 | Reason contrast | 두 짧은 문장과 승인된 연결형을 비교: `몸이 안 좋아요. 예약을 바꾸고 싶어요.` → `몸이 안 좋아서 예약을 바꾸고 싶어요.` |
| 100–110 | Multi-turn transfer | messaging_calls의 open, request, close를 포함한 5–7턴 통화 |
| 110–116 | Constraint roleplay | 제안 시간이 불가능할 때 다른 시간을 묻고 확인 |
| 116–120 | Extended Exit Check | 이유, 요청, 사과, 시간 확인을 한 번에 연결 |

`grammar_reason_eoseo`를 120분 확장에 넣을지는 강사 승인 사항이다. 승인되지 않으면 90–100분은 Vocabulary 재활용과 추가 Roleplay로 대체한다.

## 단어 자료의 슬라이드 구조

실제 Student Deck은 생성하지 않는다. 승인 후 사용할 수 있는 구조만 제안한다.

1. **Mission Word Map**
   - 상태: `아프다`
   - 행동: `바꾸다`, `취소하다`
   - 시간: `오늘`, `내일`, `아침`, `가능하다`
   - 상호작용: `연락하다`, `확인`

2. **Productive Combination Cards**
   - `예약을 바꾸다`
   - `약속을 취소하다`
   - `내일 오후는 가능하다`
   - `갑자기 연락하다`

3. **Staff Language / Receptive Support**
   - `일정 변경`
   - `예약 확인`
   - `당일 취소`
   - 학생은 번역이 아니라 다음 응답을 고른다.

4. **Q/A Prompt Cards**
   - `예약을 어떻게 하고 싶어요?`
   - `언제 가능하세요?`
   - `세 시로 바꿀까요?`
   - 각 질문은 Vocabulary Item의 `production_prompt`와 연결한다.

5. **Transfer Card**
   - 전화/메시지 시작: `안녕하세요. 갑자기 연락해서 죄송합니다.`
   - 요청: `약속을 바꾸고 싶어요.`
   - 종료: `네, 감사합니다.`

## Guided Practice

### Guided 1: Change or Cancel

교사가 상황을 제시하고 학생이 한 문장을 선택한다.

- Prompt: `내일 갈 수 없어요. Change.`
- Student: `예약을 바꾸고 싶어요.`
- Prompt: `이번 주에 갈 수 없어요. Cancel.`
- Student: `예약을 취소하고 싶어요.`

### Guided 2: Find a New Time

- Teacher: `내일 아침은 어떠세요?`
- Student: `내일 오후는 가능하세요?`
- Teacher: `네, 세 시는 가능해요.`
- Student: `네, 세 시로 바꿔 주세요.`

### Guided 3: Message / Call Frame

- Open: `안녕하세요.`
- Short state: `몸이 안 좋아요.`
- Request: `약속을 바꾸고 싶어요.`
- Apology: `갑자기 연락해서 죄송합니다.`
- Close: `감사합니다.`

`연락해서`는 고정 사과 Chunk로만 제시하고 이유 문법 분석으로 확장하지 않는다.

## 독립 Roleplay

### Roleplay A: Reschedule

- 학생은 몸 상태를 한 문장으로 말한다.
- 기존 예약을 바꾸고 싶다고 말한다.
- 상대가 제안한 시간이 불가능하면 다른 시간을 묻는다.
- 새 시간을 확인하고 감사 인사로 끝낸다.

성공 증거:

- `grammar_want_go_sipeoyo`를 사용해 변경 의도를 표현한다.
- `interaction_confirm_and_answer`로 새 시간을 확인한다.
- Core Chain의 최소 5문장을 모델 없이 사용한다.

### Roleplay B: Cancel

- 학생은 인사와 짧은 상태 문장으로 시작한다.
- 예약 또는 약속을 취소하고 싶다고 말한다.
- 당일 취소 안내를 듣고 짧게 확인 응답한다.
- 사과와 감사로 끝낸다.

성공 증거:

- `취소하고 싶어요`를 독립 산출한다.
- `당일 취소`를 듣고 적절한 짧은 응답을 한다.
- 의료 세부 정보나 긴 이유 설명 없이 상호작용을 마친다.

## Exit Check

교사는 영어 또는 상황 Prompt만 제공한다.

1. `You do not feel well.`
   - 기대: `몸이 안 좋아요.`
2. `You want to change the appointment.`
   - 기대: `예약을 바꾸고 싶어요.`
3. `Ask whether tomorrow afternoon is possible.`
   - 기대: `내일 오후는 가능하세요?`
4. `Accept 3 p.m. and request the change.`
   - 기대: `네, 세 시로 바꿔 주세요.`
5. `Apologize for contacting suddenly.`
   - 기대: `갑자기 연락해서 죄송합니다.`

판정:

- 4–5개 독립 산출: 다음 설계에서 Independent 비중 유지.
- 2–3개 독립 산출: Guided Prompt를 늘리고 Vocabulary Count 유지 여부 재검토.
- 0–1개 독립 산출: Scope를 60분 핵심 4개 Productive Item으로 축소하는 방안 검토.

## 강사가 결정해야 할 질문

1. 90분 기본, 60분 압축, 120분 확장 구성을 승인할 것인가?
2. Productive 6개, Receptive 4개, Review 4개, Homework 2개 Count를 유지할 것인가?
3. 이전 노출이 확인되지 않은 `내일`을 Review로 유지할 것인가?
4. `grammar_reason_eoseo`를 120분 확장에서만 활성화할 것인가?
5. `갑자기 연락해서 죄송합니다`를 문법 분석 없는 고정 사과 Chunk로 승인할 것인가?
6. `messaging_calls`의 open/request/close를 Transfer로 승인할 것인가?
7. Independent Roleplay에서 reschedule과 cancel을 모두 수행할지 하나를 우선할지 결정해야 한다.
